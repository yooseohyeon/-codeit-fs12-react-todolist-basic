# Level 2: 컴포넌트 분리 + 할일/완료 목록 분리

## 목표

TodoList의 CRUD 기능을 구현하되, 컴포넌트를 분리하고 할일 목록과 완료 목록을 나눠서 보여줍니다.

## 사용 개념

- `useState`
- 컴포넌트 분리
- `props`

## 데이터 구조

```js
{
  id: 1,
  title: "할일",
  completed: false,
  createdAt: "2026-03-18T12:00:00.000Z"
}
```

## 요구사항

### 1. 할일 추가 (Create)

- 텍스트 입력창과 "추가" 버튼이 있어야 합니다
- 입력창에 할일을 입력하고 "추가" 버튼을 클릭하면 새로운 할일이 목록에 추가됩니다
- 추가 후 입력창은 비워져야 합니다
- 빈 문자열은 추가할 수 없어야 합니다

### 2. 할일 목록 / 완료 목록 분리 표시 (Read)

- **할일 목록**: `completed`가 `false`인 항목만 표시
- **완료 목록**: `completed`가 `true`인 항목만 표시
- 각 섹션에 제목(`h2`)이 있어야 합니다 (예: "할일 목록", "완료 목록")
- 각 할일에는 제목과 생성 시간이 보여야 합니다

### 3. 완료 처리 / 취소 (Update)

- 할일 목록의 항목에는 "완료" 버튼이 있어야 합니다
- "완료" 버튼을 누르면 해당 항목이 완료 목록으로 이동합니다
- 완료 목록의 항목에는 "취소" 버튼이 있어야 합니다
- "취소" 버튼을 누르면 해당 항목이 다시 할일 목록으로 이동합니다

### 4. 할일 삭제 (Delete)

- 각 할일에 "삭제" 버튼이 있어야 합니다
- "삭제" 버튼을 누르면 해당 할일이 목록에서 제거됩니다

## 컴포넌트 분리 가이드

최소 아래 컴포넌트로 분리해야 합니다. 파일은 `src/components/` 폴더에 만듭니다.

| 컴포넌트 | 역할 | 필요한 props |
|---|---|---|
| `TodoForm` | 할일 입력 폼 | `onAdd` |
| `TodoItem` | 할일 항목 하나 | `todo`, `onToggle`, `onDelete` |
| `TodoList` | 할일 목록 (여러 TodoItem) | `todos`, `onToggle`, `onDelete` |

- 상태(`todos`)는 `App.jsx`에서 관리합니다
- 자식 컴포넌트에 함수를 props로 전달하여 상태를 변경합니다

## 스타일링

- CSS Module(`App.module.css`)을 사용합니다
- `App.module.css`에 `.container`와 `.completed` 클래스가 제공되어 있습니다
- 각 컴포넌트별 CSS Module을 만들어도 되고, 하나로 사용해도 됩니다
- 추가 스타일링은 자유롭게 해도 되고, 안 해도 됩니다

## 힌트

- `todos.filter(todo => !todo.completed)` 로 미완료 항목을 걸러낼 수 있습니다
- props로 함수를 넘길 때: `<TodoItem onDelete={handleDelete} />`
- 자식에서 호출할 때: `props.onDelete(todo.id)`

## 파일 구조 예시

```
src/
├── App.jsx              # 상태 관리, 전체 레이아웃
├── App.css              # 글로벌 스타일 (제공됨)
├── App.module.css       # CSS Module (제공됨)
└── components/
    ├── TodoForm.jsx     # 입력 폼
    ├── TodoItem.jsx     # 할일 항목 하나
    └── TodoList.jsx     # 할일 목록
```
